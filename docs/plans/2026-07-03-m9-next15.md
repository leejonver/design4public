### Task 3: Make `createServerSupabase()` async and propagate `await` (the codemod's blind spot)

The codemod rewrites direct `cookies()` calls but **cannot** see that `createServerSupabase()` wraps `cookies()` and is re-exported, nor propagate `await` to its 33 call sites. This is the load-bearing manual task.

**Files:**
- Modify: `lib/supabase/server.ts` (full rewrite below)
- Modify: `lib/auth.ts:28`
- Modify: all 16 `app/api/admin/**/route.ts` files that call `createServerSupabase()` (33 call sites total; see Step 3)
- Test: `npx tsc --noEmit`

**Interfaces:**
- Produces: `createServerSupabase(): Promise<SupabaseClient<Database>>` — every caller must `await` it.
- Consumes: `cookies()` from `next/headers`, now returning `Promise<ReadonlyRequestCookies>` in Next 15.

- [ ] **Step 1: Rewrite `lib/supabase/server.ts` to await cookies**

Replace the file contents with:

```ts
// Server Supabase client bound to the request cookies (@supabase/ssr).
// Used in Route Handlers and Server Components for authenticated, RLS-scoped access.
import 'server-only'
import { cookies } from 'next/headers'
import { createServerClient } from '@supabase/ssr'
import type { Database } from '../database.types'

export async function createServerSupabase() {
  const cookieStore = await cookies()
  return createServerClient<Database>(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return cookieStore.getAll()
        },
        setAll(cookiesToSet) {
          try {
            cookiesToSet.forEach(({ name, value, options }) =>
              cookieStore.set(name, value, options),
            )
          } catch {
            // Called from a Server Component — cookie writes are handled by middleware.
          }
        },
      },
    },
  )
}
```
