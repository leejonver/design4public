### 🟡 중간 위험 (부분적 작동 가능)

#### 1. RLS(Row Level Security) 미설정
**위험:**
- 모든 사용자가 모든 데이터 접근 가능
- 권한 관리 무용지물

**영향:**
- 보안 문제
- 데이터 무단 수정/삭제 가능

**해결책:**
```sql
-- 예시: 승인된 사용자만 프로젝트 읽기
CREATE POLICY "Approved users can read projects"
ON projects FOR SELECT
TO authenticated
USING (
  EXISTS (
    SELECT 1 FROM profiles
    WHERE profiles.id = auth.uid()
    AND profiles.status = 'approved'
  )
);
```

---

#### 2. 파일 업로드 미구현
**위험:**
- 이미지 업로드 불가
- 프로젝트/아이템/브랜드에 이미지 첨부 불가

**영향:**
- 텍스트 데이터는 CRUD 가능
- 시각적 콘텐츠 없음

**해결책:**
1. Supabase Storage 버킷 생성
2. 공개 정책 설정
3. `/api/upload` 엔드포인트 테스트
