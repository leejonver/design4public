## 🔍 문제 해결

### 문제 1: 로그인 실패

**증상:**
```
"로그인에 실패했습니다" 에러
또는
"사용자 정보를 가져올 수 없습니다" 에러
```

**원인:**
1. 환경 변수 미설정
2. Supabase 사용자 계정 미생성
3. 프로필 승인 상태 문제

**해결책:**

```sql
-- 1. Supabase Dashboard → SQL Editor에서 실행

-- 사용자 확인
SELECT * FROM auth.users WHERE email = 'design4public@gmail.com';

-- 프로필 확인
SELECT * FROM profiles WHERE email = 'design4public@gmail.com';

-- 프로필이 없다면 생성
INSERT INTO profiles (id, email, name, role, status)
VALUES (
  'auth-user-id',  -- auth.users의 id
  'design4public@gmail.com',
  'Design4Public',
  'master',
  'approved'
);
```

---

### 문제 2: 데이터 조회 실패

**증상:**
```
프로젝트 목록이 비어있음
또는
"relation does not exist" 에러
```

**해결책:**

1. **테이블 생성 확인:**
   ```
   Supabase Dashboard → Table Editor
   - projects 테이블 있는지 확인
   - items 테이블 있는지 확인
   - brands 테이블 있는지 확인
   ```

2. **마이그레이션 실행:**
   ```sql
   -- migrations/create_project_with_relations.sql 내용을
   -- Supabase SQL Editor에서 실행
   ```

3. **RLS 정책 확인:**
   ```sql
   -- 읽기 권한 정책 추가
   CREATE POLICY "Allow authenticated users to read projects"
   ON projects FOR SELECT
   TO authenticated
   USING (true);
   ```

---

### 문제 3: 이미지 업로드 실패

**증상:**
```
이미지 업로드 시 에러
```

**해결책:**

1. **Storage 버킷 생성:**
   ```
   Supabase Dashboard → Storage → "New bucket"
   - 이름: images
   - Public: Yes
   ```

2. **정책 설정:**
   ```sql
   -- 업로드 권한
   CREATE POLICY "Allow authenticated uploads"
   ON storage.objects FOR INSERT
   TO authenticated
   WITH CHECK (bucket_id = 'images');
   
   -- 읽기 권한 (공개)
   CREATE POLICY "Allow public reads"
   ON storage.objects FOR SELECT
   TO public
   USING (bucket_id = 'images');
   ```
