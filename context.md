# Design4Public CMS - Context Documentation

## 프로젝트 개요
design4public 공공조달 가구 납품 프로젝트 사례를 소개하는 콘텐츠 사이트의 Contents Management System 프론트엔드 개발

## 기술적 요구사항
- 반응형으로 제작하되, 데스크탑화면에 최적화하여 개발
- Ant Design React 디자인 시스템 기반 개발
- 프레임워크: Next.js 최신버전
- 백엔드 엔지니어가 기획서없이 화면과 코드를 확인하며 개발할 수 있도록 적절한 주석처리와 알기 쉬운 구조로 개발
- 모든 화면이 더미데이터와 함께 충분히 제대로 구현되어 보여질 수 있도록 제작 (더미데이터는 충분히 많이 만듭니다)

## 웹페이지 URL 구조
- Home URL: cms.design4public.com
- `/login`: 관리자 로그인페이지 (이메일과 비밀번호 입력)
- `/signup`: 이메일/비밀번호 입력. 가입 후 "이메일 인증 후 관리자 승인이 필요합니다" 안내
- `/projects`: 프로젝트 데이터 리스트뷰
- `/projects/[project_id]`: 프로젝트 데이터 상세뷰
- `/projects/new`: 프로젝트 데이터 추가 페이지 (입력폼)
- `/items`: 아이템(가구) 데이터 리스트뷰
- `/items/[item_id]`: 아이템 데이터 상세뷰
- `/items/new`: 아이템 데이터 추가 페이지 (입력폼)
- `/brands`: 브랜드(제조사) 데이터 리스트뷰
- `/brands/[brand_id]`: 브랜드 데이터 상세뷰
- `/brands/new`: 브랜드 데이터 추가 페이지 (입력폼)
- `/tags`: 태그 데이터 관리페이지, 사용하는 태그 데이터를 CRUD 할 수 있습니다
- `/managers`: CMS 사이트의 관리자들을 승인, 삭제, 권한변경 할 수 있습니다

## 레이아웃 및 디자인
- 좌측 사이드바에 GNB가 위치
- 사이드바 상단에 사이트명 (design4public 콘텐츠관리자)
- 하단부에 로그인된 사용자프로필과 로그아웃버튼 배치
- GNB에는 '프로젝트', '아이템', '브랜드', '태그', '관리자' 5개의 메뉴
- 각 메뉴 옆에 뱃지형태로 레코드의 수 표시
- 서체: Pretendard

## 콘텐츠 입력 항목

### 프로젝트
- 프로젝트명
- 프로젝트설명
- 프로젝트지역
- 완공연도
- 면적(m²)
- 프로젝트 사진: 여러장 업로드 가능, 대표이미지 지정
- 프로젝트 태그: DB 태그 중 선택하여 다중 입력
- 연결된 아이템: DB 아이템 중 검색선택
- 문의URL: tally나 구글폼 같은 웹페이지 URL
- 발행상태: 초안, 게시, 숨김 옵션

### 아이템
- 아이템명
- 아이템설명
- 아이템이미지: 여러장 업로드, 대표이미지 지정
- 나라장터URL: 나라장터종합쇼핑몰 구매 페이지 URL
- 브랜드: DB 브랜드 중 선택
- 아이템 태그: DB 태그 중 선택하여 다중입력
- 상태: 구입가능, 단종, 숨김 중 선택

### 브랜드
- 브랜드명
- 브랜드설명
- 브랜드로고 이미지: 로고이미지를 원형으로 크롭하여 업로드
- 브랜드커버 이미지: 브랜드 커버이미지 1장 업로드
- 브랜드URL: 브랜드 홈페이지 URL
- 상태: 노출, 숨김 중 선택

### 태그
- 태그명

### 관리자 (마스터 권한만 접근가능)
- 관리자명: 관리자이름 입력,수정
- 권한: 마스터, 관리자, 컨텐츠매니저 중 선택
- 승인상태: 가입요청 상태 레코드에 대해 승인처리

## 개발 원칙
- 모든 코드에 적절한 주석 작성
- 컴포넌트 구조를 직관적으로 유지
- 더미데이터를 풍부하게 제공하여 QA 용이성 확보
- 백엔드 개발자가 쉽게 이해할 수 있는 구조 유지
